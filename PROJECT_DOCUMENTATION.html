1. Project Overview

HireFlow is an AI-powered mock interview platform. A candidate uploads a resume, then the AI conducts a structured 15-question voice interview tailored to that resume. At the end, the candidate gets a printable performance report.

Core idea: Replace the awkwardness of mock interviews with a friendly AI that adapts to each candidate's background, gives realistic questions, and provides honest feedback — without needing a human interviewer.

Target Users

  • Students preparing for placements (B.Tech, MBA, etc.)
  • Working professionals practicing for new roles
  • Self-learners building interview confidence
  • HR teams who want low-cost preliminary screening

5. End-to-End User Flow

  1. Open the site → React loads, user lands on setup page
  2. Upload resume (PDF/DOCX/TXT) or paste text → Backend extracts via pdf-parse / pdfjs / mammoth
  3. Enter target role + tick "Enable voice mode" → pick AI voice persona
  4. Click "Start interview" → Backend creates session, asks Groq Llama 3.3 70B for Q1 → returns text
  5. AI speaks Q1 → "Hi [Name], great to meet you. To know more about you, please tell me a little about yourself."
  6. After AI finishes (auto-detected by polling speechSynthesis.speaking) → mic auto-opens, orb turns green
  7. Candidate speaks → MediaRecorder captures audio @ 64kbps Opus + Web Speech API shows live preview
  8. 5s of silence → MediaRecorder stops, audio sent to /api/transcribe → Groq Whisper returns transcript → Groq Llama 8B fixes typos
  9. Cleaned transcript sent to /api/interview/answer → Backend appends to session history, sends to Llama 3.3 70B with system prompt + question count → receives next question
  10. Question shown in chat + AI speaks it → loop continues to question 15
  11. After 15th answer → Backend tells AI it's the final answer → AI returns structured feedback ending with [INTERVIEW_END]
  12. "Interview complete" page → printable HTML report with the feedback → user clicks "Print / Save as PDF"

Session Persistence

Sessions are stored as JSON in %TEMP%/ai-interview-sessions.json. After every /api/interview/answer response, the session's chat history is saved. On backend restart, all sessions are loaded — interviews survive crashes/restarts.

Why os.tmpdir() not the project folder?

Bug we fixed: writing session JSON to the backend folder caused node --watch to detect the file change and restart the server in an infinite loop. Storing in OS temp dir is outside the watch scope.

8. Voice Pipeline

Speech Synthesis (AI → Speakers)

  1. Browser's window.speechSynthesis API used
  2. 7 personas mapped to system voices: Aria, Sophia, Zira (female) / Guy, Ethan, David, Marcus (male)
  3. Each persona has a custom pitch (Marcus: 0.6 = deep, Sophia: 1.15 = light)
  4. Text prepended with "... " (4-char silence) to absorb Chrome's first-word clipping bug
  5. A silent warm-up utterance plays first to wake the audio engine
  6. Mic only opens after polling confirms speaking=false for 400ms + 200ms buffer

Speech Recognition (Candidate → Text)

Dual capture:

  • Web Speech API (browser STT) — shows live transcript preview while you talk
  • MediaRecorder (Opus @ 64kbps) — captures actual audio in parallel
  • After 5s of silence (no transcript growth) → audio sent to Groq Whisper for final accurate transcript
  • Groq Llama 3.1 8B fixes typos only (preserves all words)
  • Cleaned text becomes the submitted answer
Why dual capture? Web Speech is fast but inaccurate (Google's STT struggles with accented English and tech terms). Whisper is accurate but only runs on submission. Using both gives instant visual feedback + correct final transcript.

10. AI System Prompt (Llama 3.3 70B)

The system prompt tells the AI how to conduct the interview. Key sections:

Interview Structure

  1. Q1: Greet by name → "Hi [Name], great to meet you. Tell me a little about yourself."
  2. Check coverage of 7 self-intro points (name, education, current work, skills, achievement, target role, why fit, career direction). If missing → ask one follow-up.
  3. Q2 to Q15: Resume-anchored questions cycling through 7 categories (project deep-dive, technical depth, behavioral, hypothetical, gap probe, comparison, real-world application).
  4. Final feedback: Self-intro coverage table + Strengths + Areas to Improve + Communication + Technical Depth + Overall Score /10, ending with [INTERVIEW_END] token.

Resume Grounding Rule

Every question MUST reference a specific item from the resume — a project name, technology, internship, or skill. Generic questions are forbidden.

Off-Topic Handling

If candidate answer is off-topic, gibberish, or unrelated to resume → AI calls it out live and notes it in final feedback.

Backend Enforcement

Beyond the prompt, the backend:

  • Counts user answers and forces done=true at exactly 15
  • Overrides AI's [INTERVIEW_END] if fired too early
  • Strips leaked meta-commentary like "(I'll internally track...)" via regex
  • Injects question count via dynamic system instruction (NOT user message — to avoid leaks)

11. Setup & Run Instructions

Prerequisites

  • Node.js 18 or higher
  • npm (comes with Node)
  • A Groq API key from https://console.groq.com/keys (free)

Step 1: Install dependencies

# Backend
cd backend
npm install

# Frontend
cd ../frontend
npm install

Step 2: Configure API keys

Create backend/.env:

GROQ_API_KEY=gsk_your_groq_key_here
GEMINI_API_KEY=optional_gemini_key   # not used in current build, kept for fallback
PORT=5174

Step 3: Run both servers

# Terminal 1 — backend
cd backend
node server.js

# Terminal 2 — frontend
cd frontend
npm run dev

Step 4: Open the app

Visit http://localhost:5173/ in Chrome or Edge.

Browser requirement: Voice features need Chrome or Edge. Firefox has limited Web Speech support. Mic permission must be granted.
